Purchase list service

The purchase list service is the medium size service in the example application. It manages purchase lists. A purchase list consist of a name and a due date. It also references the items inside a purchase list. This class simply consists of the name of the ingredient that should be bought.

Database H2

This service uses an H2 persistent DBMS. The database is stored under the user directory. Two files can be found there:

  • purchaseList.mv.db
  • purchaseList.trace.db
In case of an service failure, just delete the files stated above and restart the service.

The service has an database browser. The username is admin and the password is an empty text field. The browser is an simple database query service, which you can use to see the changes written to the database.

REST API Spring Data HATEOAS REST

This service exposes an REST API to see an logical view of the data stored in the database.

Under HAL Browser you have an interactive online console for accessing the REST API of this service.
